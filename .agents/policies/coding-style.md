# Coding Style Policy

## Core principles
1. Prefer clarity over cleverness.
2. Make small, focused changes.
3. Fix root causes, not symptoms.
4. Keep behavior changes explicit.

## Source code
- Follow existing formatting and linting rules.
- Use descriptive names.
- Keep functions cohesive and testable.
- Avoid speculative abstractions.

## Testing
- Add or update tests for behavior changes when practical.
- Start with targeted tests nearest to changed code.

## Documentation
- Update docs when external behavior or workflow changes.
- Keep notes concise and actionable.
